  1. 1. Personal Website & Blogs (You're looking at it right now!!) :

    The idea for this project was simple. I wanted to make a personal website unlike other CS,SE,ECE, etc. students in UW. I wanted to make one that was actually personal. The purpose of this website is to show-case the real me outside of my career and my academics. I wanted to show people what I love to do after working/studying.

    This site is created using HTML, CSS, JavaScript from scratch. I used no frameworks or libraries (with the exception of jQuery) for this site on purpose, to attain a better understanding of front-end development.

    Overall, I am pretty proud of this project because I ended up growing my knowledge of front-end development as well as a little bit of back-end development.

      Technologies I used:
    • HTML5 - content
    • CSS3 - styling
    • JavaScript & jQuery - interactivity and client-side input validation
    • PHP - form validation, processing, mailing features
    • XAMPP - Apache local server

      Main obstacles I faced:
    • Writing and debugging JS and using jQuery whilst still being fairly new to them.
    • Learning server-side development with PHP.
    • WAMP server not starting and major difficulties configuring SendMail (SMTP) (for sending e-mails through forms).

  2. 2. WIPEOUT! Automated Runway Sweeper:

    This project, I'd have to admit, was a group project for a course. Before transferring into Computer Science, I was in Mechatronics Engineering. This robotics project was for my introductory programming course GENE121 in 1A.

    What this robot does is that it sweeps debris off of a miniature model of an airport runway. It features a line-following algorithm that made use of simple programming concepts (think loops and if-statements). Intuitively, using basic (like basic basic) CS concepts, a line-following algorithm would probably feature a loop, where if the sensor sees white (a boolean value), then continue, and if the sensor sees black, then turn left/right to correct the robot's path. This algorithm turned out to be inefficient and inaccurate as the robot will end up zig-zagging. Instead, the line-following algorithm goes something like this: 1. I am going to make the light sensor see the edge of the line where it is 50% black and 50% white. 2.If I see more black than white, I will make a correction left or right based on how much more black I've seen. 3. I'll do the same if I see more white. This way, the robot will not zig-zag and will follow the line more accurately.

      Technologies I used:
    • RobotC - C/C++ based robot programming language for NXT and VEX robotics
    • [Hardware] NXT light sensors, motors, encoders, and a programmable brick microcomputer
